## Dependencies {#907b3fae8582494eaf2438c80ad63660}


#### Deb Package Manager {#8af945d2feb64dc49258460f34799b47}


```bash
echo 'deb [trusted=yes] https://repo.goreleaser.com/apt/ /' | tee /etc/apt/sources.list.d/goreleaser.list && apt-get update -y && \
apt install nfpm
```


#### Docker {#d826dd6a316240048c0401d557958ef5}


```bash
apt-get update -y && \
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | apt-key add - && \
add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" && \
apt-get update -y && apt-get install docker-ce docker-ce-cli containerd.io -y && \
docker version
```

## Interx Installation {#ddebded562ee4a1e95e8aa1bb6dcbd22}


:::note

INTERX Repo is accessible [here](https://github.com/KiraCore/interx).

:::




Assume root permissions


```bash
sudo -s
```


Set desired INTERX release version and binaries repo as env variables within `/etc/profile`, with [Bash-utils](/docs/bash-utils) or manually. Sourcing `/etc/profile` is necessary.


```bash
setGlobEnv INTERX_VERSION "v0.4.34" && \
setGlobEnv INTERX_REPO "$HOME/interx" && \
loadGlobEnvs
```


Clone repository and install


```bash
rm -fr $INTERX_REPO && rm -fr $GOBIN/interx && mkdir $INTERX_REPO && cd $INTERX_REPO && \
git clone https://github.com/KiraCore/interx.git -b $INTERX_VERSION $INTERX_REPO && \
chmod -R 777 ./scripts && make install && \
echo "SUCCESS installed interx $(interx version)" || echo "FAILED"
```
